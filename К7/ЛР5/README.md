Задача 1. Дан отсортированный список скоростей. Найти пару скоростей с заданной суммой.
Бизнес логика:  Берём два указателя : один в начале списка (минимальная скорость), второй в конце списка (максимальная скорость). Считаем их состоятельность. Если сумма: эквивалент заданной → пара найдена, возвращаем результат; меньше заданной → увеличиваем левый указатель (нужно больше); больше заданной → уменьшаем правый указатель (нужно меньше). Повторяем, пока указатели не встретятся Если не нашел — такой пары нет.
Задача 2. Проверить, является ли название модели палиндромом.
Бизнес логика: Приведите текст к единому мнению: сделать все буквы строчными, удалите пробелы и лишние символы (если нужно). Сравните сигнал с ее обратной репутацией. Если произойдет — модель палиндром , иначе — нет.
Задача 3. Список (модель, скорость) — сортировка: • по скорости; • по модели.
Бизнес логика: Сортировка по скорости Используется, когда важна проверка производительности. Упорядочиваем список по многочисленным последствиям (по старению или убыванию). Сортировка по моделям Используется для поиска и каталогизации. Упорядочиваем список по названию модели в алфавитном порядке. кистей: Получаем упорядоченный список, удобный для анализа, отчётов и интерфейса пользователя. Данные о целостности: модель всегда обеспокоена своей скоростью.
Задача 4. Поиск скорости модели
Бизнес логика: Получаем название модели, которую нужно найти. Проходимость по списку моделей: сравниваем название текущей модели с иским. Если модель найдена: возвращаем ее скорость. Если список просмотрен и совпадений нет: Сообщаем, что модель не найдена
