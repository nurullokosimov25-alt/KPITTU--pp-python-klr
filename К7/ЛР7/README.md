Задача 2. Генератор накопительной суммы. running_sum([1,2,3]) → 1, 3, 6 Требования: 
• корректная логика 
• обработка пустого списка 
• 3 unit-теста Акцент: обновляемое состояние.

Бизнес-логика решения:
•	total — аккумулятор, хранит текущую сумму.
•	На каждом шаге сумма обновляется и сразу отдаётся через yield.
•	Если список пустой — цикл не выполнится, генератор корректно завершится (ничего не вернёт).
